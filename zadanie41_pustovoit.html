Teoria dotycząca pętli

1. Pętla for

Konstrukcja:

for (zainicjowanie_zmiennej; warunek_kończący_wykonywanie_pętli; zmiana_zmiennej) {
  kod który zostanie wykonany pewną ilość razy
}

Opis: Pętla for wykonuje kod określoną liczbę razy. Najpierw inicjalizuje zmienną, następnie sprawdza warunek, wykonuje kod, zmienia zmienną i ponownie sprawdza warunek.

Schemat blokowy: Start → Inicjalizacja → Sprawdzenie warunku → (Tak) Wykonanie kodu → Zmiana zmiennej → (Powrót do sprawdzenia warunku) → (Nie) Koniec

2. Pętla while - z kontrolowanym wejściem

Konstrukcja:

while (wyrażenie_sprawdzające_zakończenie_pętli) {
  ...fragment kodu który będzie powtarzany...
}

Opis: Pętla while najpierw sprawdza warunek, potem coś wykonuje. Pętla może się nie wykonać, jeśli warunek od początku jest fałszywy.

Schemat blokowy: Start → Sprawdzenie warunku → (Tak) Wykonanie kodu → (Powrót do sprawdzenia warunku) → (Nie) Koniec

3. Pętla do...while - z kontrolowanym wyjściem

Konstrukcja:

do {
  ...fragment kodu który będzie powtarzany...
} while (warunek);

Opis: Pętla do...while zawsze wykona jedną iterację, zanim sprawdzi warunek. Zawsze więc wykona jakieś zadanie.

Schemat blokowy: Start → Wykonanie kodu → Sprawdzenie warunku → (Tak) Powrót do wykonania kodu → (Nie) Koniec
